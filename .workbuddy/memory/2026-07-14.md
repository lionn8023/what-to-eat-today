## 第二轮：品类 bug 修复 + 数量选项 + 真·地图可视化

用户反馈：烧烤/甜点/西餐永远搜不到，别的品类有结果。

**根因（关键知识点）**：高德 `AMap.PlaceSearch` 的 `type` 参数**只认 POI 分类**（6 位编码如 `050117`=火锅店，或 20 个大类中文串如 `餐饮服务`），不能传搜索词。原代码 `new AMap.PlaceSearch({ type: keyword })` 把关键词当类型，导致非法 type 静默把结果过滤成空——火锅/快餐/咖啡/日料 因类型路径含这些字或关键词本身能搜到才"侥幸"有结果，烧烤/甜点/西餐被卡死。

**修复**：
- `amap.js` 的 `runPlaceSearch`：`type` 固定为 `'餐饮服务'`（锁定餐饮大类），具体品类交回 `searchNearBy(keyword)` 做文本检索。
- `SearchPanel.jsx` 品类 value：甜点 `'面包甜点'`→`'甜品'`，西餐 `'西餐'`→`'西餐厅'`（更稳的搜索词）；烧烤保持 `'烧烤'`（去掉 type 干扰后即正常）。
- 店铺数量 `LIMIT_OPTIONS` 改为 10/20/30/50（去掉 100），自定义输入上限同步 100→50；`App.jsx` 恢复时 `limit` 也 `Math.min(…,50)` 钳制。

**新功能 真·地图可视化**：
- 新建 `src/components/MapView.jsx` + `MapView.module.css`：加载 AMap 地图，把 shops 渲染成 `AMap.Marker`，`setFitView` 自动框住全部；响应 `activeShop` 用 `map.setZoomAndCenter(16, pos)` 平滑"飞过去"并弹 `AMap.InfoWindow`（名称/地址/评分/人均）。点击 Marker 回调 `onSelectShop`。地图加载失败（无 Key）时优雅显示提示而非白屏。
- `App.jsx` 引入 MapView，布局改为左右两栏（`grid 380px 1fr`，左 SearchPanel sticky，右 地图+列表），窄屏 `<880px` 自动单列。`activeShop={selectedShop}` 接入，转盘/列表/Marker 选中均触发镜头飞行。
- `App.module.css` 增加 `.leftCol/.rightCol` 及响应式。

验证：`npm run build` 通过（48 modules）。dev server（:5173）HMR 已热更，用户可在预览中实测。地图实际渲染需在浏览器查看（本环境无法跑浏览器）。

## 第三轮：转盘遮挡修复 + 镜头飞行 + 去掉数量上限 + 街景/导航

用户截图反馈四点问题，全部修复：

1. **转盘被地图遮挡（z-index 穿透）**：`Wheel.module.css` overlay `z-index` 从 100→9999；`ShopDetail.module.css` 同步 200→9999。高德内部控件版权文字不再穿透到弹窗上层。
2. **镜头飞过去无效果**：根因是 Wheel 揭晓赢家时没有通知地图。新增 `onHighlight(shop)` 回调链路——Wheel 转完一停立刻调用；App 新增 `mapFocusShop` 独立状态（与 `selectedShop` 详情弹窗解耦）；MapView 监听 `activeShop={mapFocusShop || selectedShop}` 触发 setZoomAndCenter + InfoWindow。现在流程：转盘停 → 地图立刻飞到赢家位置 → 用户关闭转盘后地图已就位。
3. **去掉自定义数量 50 上限**：`SearchPanel.jsx` 自定义输入校验从 `num <= 50` 改为只要求 `>= 1`；`App.jsx` 初始化和恢复也去掉 `Math.min(…,50)` 钳制。（高德 API 单次 pageSize 最大 50、支持分页，代码已自动分页拉取，UI 不再人为限制。）
4. **街景/导航功能**：`ShopDetail.jsx` 新增双按钮操作区——「🚗 导航到店」用高德 URI Scheme 跳驾车导航页（`callnative=1` 支持手机端唤起 App）、「🚗 查看街景」打开高德街景页面。原有「在高德地图中打开」保留在下方。配套 `.actionRow / .actionBtn` CSS（grid 双列等宽、新拟态硬阴影风格统一）。

验证：`npm run build` 通过（48 modules，CSS 14.04 kB / JS 170.30 kB）。

## 第五轮：修复「刷新后省份/城市又被置为全国」+「定位应填入正确省市」

用户反馈：刷新后省份/城市又被重置为全国；点击定位时应按正确位置填入省/市。

**根因**：`handleLocate` 把 `province` 写成 `NATIONAL.name`（全国）、`city` 写 `''`，导致只要用过"定位"，`localStorage` 存的 province 就是"全国"，刷新自然也是全国；而 `lngLatToAddress` 仅返回地址字符串、未把省/市结构化出来，定位时也无从填正确省市。

**修复**（src/services/geocoder.js、src/components/SearchPanel.jsx）：
- `geocoder.lngLatToAddress` 改为返回 `{ formattedAddress, province, city }`（从 `regeocode.addressComponent` 解析，处理直辖市 `city` 为数组/空的情况）。
- `handleLocate` 据此 `setProvince(foundProvince)`/`setCity(foundCity)`；直辖市（city 与 province 同名）`foundCity` 置空。
- 新增 `MUNICIPALITY_NAMES`（京/津/沪/渝）：直辖市不自动选第一个区、城市下拉禁用并显示自身（如"北京市"）。
- 省份列表加载/失败的兜底：合并 `initialProvince`，确保刷新即使 AMap 城市数据不全也能正确显示已保存省份。

验证：`npm run build` 通过（48 modules）。dev server 重启于 :5173。

注：开发服务器会偶发退出，需 `npm run dev` 重启（后台任务）。
